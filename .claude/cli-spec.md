# ELAP CLI — Build Specification

Canonical build specification for the ELAP command-line tool. The implementation
must match this exactly: same architecture, files, CLI surface, behavior, message
strings, and safety logic. Where exact strings or code idioms are given, reproduce
them verbatim.

---

Build a macOS command-line tool called ELAP (Extended Laptop) that fully disables the MacBook's built-in display while the lid stays open — a true disconnect from WindowServer compositing (the GPU stops rendering to it), not a backlight dim. It re-enables on demand and ships with safety guards so the user can never end up with a blank screen.

The mechanism is private SkyLight.framework APIs resolved at runtime via dlsym. SIP does not need to be disabled — these calls work from user space. Acknowledge in comments that the tool will break on future macOS releases that rename or remove the symbols.

### 2.1 Private-API typealias — the critical detail

```swift
typealias CGSSetDisplayEnabledFn = @convention(c) (CGDisplayConfigRef, CGDirectDisplayID, Bool) -> Int32
```

CRITICAL, non-obvious: the first argument is a `CGDisplayConfigRef` obtained from the public `CGBeginDisplayConfiguration()` — NOT a `CGSConnectionID` integer. `CGSConfigureDisplayEnabled` is a private complement to the standard CoreGraphics display-configuration transaction (`CGBeginDisplayConfiguration` → changes → `CGCompleteDisplayConfiguration`). Passing an integer would be dereferenced as a pointer and crash. Returns 0 on success, non-zero on failure.

### 2.2 Symbol loader — struct SkyLightAPI

Stores `let _setEnabled: CGSSetDisplayEnabledFn` and `let symbolName: String`.

`func setEnabled(_ displayID: CGDirectDisplayID, _ enabled: Bool) throws` — wraps the private call in the public transaction:
1. `var config: CGDisplayConfigRef?`; `var err = CGBeginDisplayConfiguration(&config)`. Guard `err == .success` AND unwrap `let config = config`; otherwise throw `DTError.apiFailure("CGBeginDisplayConfiguration", err.rawValue)`.
2. `let ret = _setEnabled(config, displayID, enabled)`. If `ret != 0`, call `CGCancelDisplayConfiguration(config)` and throw `DTError.apiFailure(symbolName, ret)`.
3. `err = CGCompleteDisplayConfiguration(config, .permanently)`. Guard `err == .success`; else throw `DTError.apiFailure("CGCompleteDisplayConfiguration", err.rawValue)`. `.permanently` survives process exit (reverts on logout/reboot).

`static func load(verbose: Bool = false) throws -> SkyLightAPI`:
- Framework path: `"/System/Library/PrivateFrameworks/SkyLight.framework/SkyLight"`.
- Build `var handles: [UnsafeMutableRawPointer?] = []` in priority order: (1) `if let h = dlopen(path, RTLD_LAZY | RTLD_NOLOAD) { handles.append(h) }` (already-mapped check); (2) `if let h = dlopen(path, RTLD_LAZY) { handles.append(h) }` (fresh load); (3) unconditionally `handles.append(dlopen(nil, RTLD_LAZY))` (global symbol table, last resort).
- Helper: `func find(_ name: String) -> UnsafeMutableRawPointer? { handles.lazy.compactMap { dlsym($0, name) }.first }` — lazy short-circuit on first non-nil dlsym.
- Candidate symbols, tried in this exact order (keep the explanatory comment about macOS versions): `"CGSConfigureDisplayEnabled"` (macOS 13–15 Ventura/Sonoma/Sequoia, primary), `"CGSSetDisplayEnabled"` (pre-Ventura fallback), `"SLSConfigureDisplayEnabled"` (SkyLight-prefix variant, internal/debug builds), `"SLSSetDisplayEnabled"` (SkyLight-prefix variant).
- For the first hit: if verbose, `print("[verbose] Using symbol: \(name)")`; `let fn = unsafeBitCast(sym, to: CGSSetDisplayEnabledFn.self)`; `return SkyLightAPI(_setEnabled: fn, symbolName: name)`.
- If none resolve, `throw DTError.symbolNotFound(candidates)`.

### 2.3 Errors — enum DTError: Error

Cases: `noBuiltInDisplay`, `noExternalDisplay`, `symbolNotFound([String])`, `apiFailure(String, Int32)`. Conform to `LocalizedError` with `errorDescription`:
- `noBuiltInDisplay` → `"No built-in display detected on this machine."`
- `noExternalDisplay` → multi-line: states no active external display found; refuses to disable the built-in because that leaves a blank screen; says connect a monitor and retry; notes `(Use --force to bypass this check.)`
- `symbolNotFound(names)` → lists the tried symbols (each indented two spaces), prints `ProcessInfo.processInfo.operatingSystemVersionString` as macOS `<ver>`, and suggests: `nm /System/Library/PrivateFrameworks/SkyLight.framework/SkyLight | grep -iE "display|enabled"`.
- `apiFailure(sym, code)` → `"Private API '\(sym)' returned error code \(code)."`

### 2.4 Display model, state file & discovery

```swift
struct DisplayInfo { 
  let id: CGDirectDisplayID
  let isBuiltIn: Bool
  let isActive: Bool
  let bounds: CGRect
}
```

`isActive` = currently composited by WindowServer; `bounds` = native resolution in points.

State-file persistence (a disabled display vanishes from `CGGetOnlineDisplayList`, so we persist its ID). Module-level path computed via an immediately-invoked closure:

```swift
private let stateFilePath: String = {
    FileManager.default.homeDirectoryForCurrentUser
        .appendingPathComponent(".elap-builtin-id").path
}()
```

Functions: `saveBuiltInDisplayID(_ id:)` writes `String(id)` atomically (UTF-8); `clearBuiltInDisplayID()` removes the file; `loadSavedBuiltInDisplayID() -> CGDirectDisplayID?` reads, trims whitespace/newlines, and parses to `CGDirectDisplayID?`. All use `try?` (best-effort).

`func fetchDisplays(verbose: Bool = false) -> [DisplayInfo]`:
1. Call `CGGetOnlineDisplayList(0, nil, &count)`, then again with an allocated `[CGDirectDisplayID]` to fill it. Map each ID to a `DisplayInfo` using `CGDisplayIsBuiltin(id) != 0`, `CGDisplayIsActive(id) != 0`, `CGDisplayBounds(id)`. If verbose, `print("[verbose] Display \(id): <built-in|external>, <active|INACTIVE>, <W>x<H>")`.
2. Recovery, only if `!infos.contains(where: { $0.isBuiltIn })` (the built-in was disabled and dropped from the list):
  - Strategy 1 (primary): loop `for probe: CGDirectDisplayID in 1...32` calling `CGDisplayIsBuiltin(probe)` (queries hardware, works on disabled displays); take the first hit and break immediately.
  - Strategy 2 (fallback): only if `recoveredID == nil` after the probe, use `loadSavedBuiltInDisplayID()`.
  - If recovered, append `DisplayInfo(id: id, isBuiltIn: true, isActive: false, bounds: .zero)` (note `isActive: false, bounds: .zero` — it is invisible). Verbose messages note "found via ID probe" / "found via state file".

### 2.5 Signal handlers (re-enable on Ctrl+C / SIGTERM)

C callbacks can't capture Swift closures, so use module-level globals: `private var _sigDisplayID: CGDirectDisplayID = 0` and `private var _sigSetFn: CGSSetDisplayEnabledFn? = nil`.

`func installReenableHandlers(api: SkyLightAPI, displayID: CGDirectDisplayID)` stores the globals, then installs `let handler: @convention(c) (Int32) -> Void` for both SIGINT and SIGTERM. The handler receives the signal number as its sole parameter `sig` and: `if _sigSetFn is set`, opens a fresh `CGBeginDisplayConfiguration(&cfg)` transaction, calls `fn(cfg, _sigDisplayID, true)` (re-enable), `CGCompleteDisplayConfiguration(cfg, .permanently)`; then restores the default with `signal(sig, SIG_DFL)` and re-raises via `kill(getpid(), sig)` so the shell sees the correct exit status.

`func removeReenableHandlers()` resets SIGINT and SIGTERM to `SIG_DFL`.

### 2.6 Countdown helpers

- `func countdownSeconds(_ n: Int)`: for `i` from `n` down through `1`, `print("\r  \(i)...", terminator: "")`, `fflush(stdout)`, `Thread.sleep(forTimeInterval: 1.0)`. Afterward erase the line: `print("\r\u{1B}[2K", terminator: "")` then `fflush(stdout)` (CR, then ESC `\u{1B}`, then `[2K` = CSI erase-entire-line; no newline).
- `func waitForEnterOrTimeout(seconds: Int) -> Bool`: return `false` if `seconds <= 0`. Use a `var entered = false` guarded by an `NSLock`. Spawn `Thread.detachNewThread { _ = readLine(); mu.lock(); entered = true; mu.unlock() }`. On the main thread, loop `elapsed` over `0..<seconds`, printing `"\r  Auto-reverting in \(remaining)s... (Press Enter to keep disabled)"` (`+fflush`), sleeping 1s, then checking the locked flag; if set, erase the line with `\r\u{1B}[2K` and return `true` (user kept it disabled). After the loop, do a final locked check, `print("")` (newline), and return the flag. So: Enter-before-deadline → erases line, returns true; timeout → prints a newline, returns false. The function owns all line management; callers do not erase.

### 2.7 Core disable logic — func disableBuiltInDisplay(force: Bool, timeout: Int, verbose: Bool) throws

Shared by `off` and `toggle`:
1. `let api = try SkyLightAPI.load(verbose:)`; `let displays = fetchDisplays(verbose:)`. No built-in → throw `.noBuiltInDisplay`.
2. If `!builtin.isActive` → `print("Built-in display is already disabled.")` and return.
3. External-display guard (skipped iff `force`): `if !force { guard displays.contains { !$0.isBuiltIn && $0.isActive } else { throw .noExternalDisplay } }`.
4. Countdown warning (skipped iff `force`): `if !force { print("Warning: About to disable the built-in display."); print("  To re-enable at any time: elap on"); print("  Press Ctrl+C to cancel"); countdownSeconds(5) }`.
5. `saveBuiltInDisplayID(builtin.id)` — before disabling (the only way to find it after it vanishes from the online list).
6. `try api.setEnabled(builtin.id, false)`.
7. `installReenableHandlers(api:, displayID:)` — installed immediately, regardless of timeout.
8. Immediately `print("Built-in display disabled. GPU is no longer rendering it.")` and `print("To re-enable: elap on")`.
9. `if timeout > 0`: `let confirmed = waitForEnterOrTimeout(seconds: timeout)`, then `removeReenableHandlers()`. If `!confirmed` (timeout): `print("Timeout elapsed. Re-enabling built-in display...")`, `try api.setEnabled(builtin.id, true)`, `clearBuiltInDisplayID()`, `print("Built-in display re-enabled.")`. Else (Enter pressed): `print("Confirmed. Built-in display stays disabled.")`. If `timeout == 0`, the handlers stay installed for the process's life (intentional for scripts; SIGKILL can't be caught so it leaves the display off).

### 2.8 CLI commands (ArgumentParser)

Root struct `ELAPCli: ParsableCommand` with `CommandConfiguration`:
- `commandName`: `"elap"`
- `abstract`: `"Toggle the macOS built-in display on/off while keeping the lid open."`
- `discussion`: explains it uses private SkyLight/CoreGraphics APIs to disconnect the built-in display, freeing the GPU; not a public API and may break on future macOS; and "Exit codes: 0 = success, 1 = error, 2 = no external display found."
- `subcommands`: `[List.self, Status.self, On.self, Off.self, Toggle.self, Watch.self, Daemon.self]`

Each command is a nested struct `…: ParsableCommand` with its own `CommandConfiguration(abstract:)`. Commands that take verbose declare `@Flag(name: .shortAndLong, help: "Show verbose debug output.") var verbose = false` (gives `-v/--verbose`). Use these exact abstracts:

- **list** — abstract `"List all connected displays with IDs, resolutions, and status."` Has verbose. Prints `"No displays found."` if empty; else for each `(i, d)` in `enumerated()`:
  - `let kind = d.isBuiltIn ? "Built-in Retina" : "External      "` (note: "External" padded with spaces to align)
  - `let res = "<W>x<H>"`
  - `let state = d.isActive ? "ENABLED" : "DISABLED"`
  - then `print("  Display \(i + 1): \(kind)  \(res.padding(toLength: 10, withPad: " ", startingAt: 0))  ID: \(d.id)  [\(state)]")`. Numbering starts at 1 (`i + 1`).

- **status** — abstract `"Show whether the built-in display is currently enabled or disabled."` Has verbose. If no built-in: `printErr(.noBuiltInDisplay)`, `throw ExitCode(1)`. Else `print("Built-in display (ID: \(b.id)): \(b.isActive ? "ENABLED" : "DISABLED")")`. (This Status type is the top-level status command — distinct from the nested `Daemon.Status` below.)

- **on** — abstract `"Enable the built-in display."` Has verbose. Load API, find built-in (throw `.noBuiltInDisplay` if absent). If `b.isActive`: `print("Built-in display is already enabled.")` and return. Else `try api.setEnabled(b.id, true)`, `clearBuiltInDisplayID()`, `print("Built-in display re-enabled.")`. Catch `DTError/Error` → `printErr + throw ExitCode(1)`.

- **off** — abstract `"Disable the built-in display (frees GPU from rendering it)."` Flags: `@Flag(help: "Skip the 5-second safety countdown and the external-display guard.") var force = false` (no `name:` → long-only `--force`, no `-f`); `@Option(help: ArgumentHelp("Auto-re-enable after N seconds if the user does not confirm. 0 = no auto-revert.", valueName: "seconds")) var timeout: Int = 0`; plus verbose. Body: `try disableBuiltInDisplay(force:, timeout:, verbose:)` (passing verbose through). Catch `.noExternalDisplay` → `throw ExitCode(2)`; other `DTError/Error` → `throw ExitCode(1)` (all via `printErr`).

- **toggle** — abstract `"Toggle the built-in display: disable if enabled, enable if disabled."` Flags: `@Flag(help: "Skip the 5-second safety countdown when disabling.") var force = false` (long-only `--force`); same `timeout` `@Option`; plus verbose. Body: fetch displays, find built-in. If `b.isActive`: `try disableBuiltInDisplay(force:, timeout:, verbose:)`. Else: load API, `try api.setEnabled(b.id, true)`, `clearBuiltInDisplayID()`, `print("Built-in display enabled.")`. Same exit-code mapping as `off`. Note: although `toggle`'s `--force` help text mentions only the countdown, `force` is threaded into `disableBuiltInDisplay`, so it bypasses both the countdown and the external-display guard — identical behavior to `off --force`; only the help string differs.

- **watch** — abstract `"Watch for display changes; auto-enable built-in when all external displays disconnect."` Has verbose. Uses module-level globals `private var _watchAPI: SkyLightAPI? = nil` and `private var _watchVerbose: Bool = false` (C callback can't capture). Body: set `_watchAPI = try SkyLightAPI.load(verbose:)` and `_watchVerbose = verbose` (catch `DTError` → `printErr + ExitCode(1)`). Print two lines: `"Watching display changes. Built-in will be enabled automatically"` then `"when all external displays disconnect. Press Ctrl+C to stop."` Register `CGDisplayRegisterReconfigurationCallback({ _, flags, _ in … }, nil)` then `RunLoop.main.run()`. The callback (only middle param `flags` used): `guard !flags.contains(.beginConfigurationFlag) else { return }` (fires twice — act only post-change); `guard flags.contains(.removeFlag) else { return }`; if `_watchVerbose` `print "[watch] External display removed, checking state…"`; `fetchDisplays`; `guard !displays.contains(where: { !$0.isBuiltIn && $0.isActive }) else { return }` (an external is still active); find built-in (`else { return }`); if it's already active, optionally `print "[watch] No external displays; built-in already enabled."` and return; otherwise `print("All external displays disconnected — enabling built-in display…")`, `try api.setEnabled(builtin.id, true)`, `clearBuiltInDisplayID()`, `print("Built-in display enabled.")` (on error, `printErr`).

- **daemon** — abstract `"Manage the background watch daemon (auto-enables built-in on external disconnect)."` subcommands: `[Install.self, Uninstall.self, Status.self]`. Static `agentLabel = "com.elap.watch"` and a static computed `plistPath`:
  ```swift
  FileManager.default.homeDirectoryForCurrentUser
      .appendingPathComponent("Library/LaunchAgents/\(agentLabel).plist").path
  ```

  - **daemon install** — abstract `"Install and start the watch daemon as a login agent."` Resolve the running binary's real path: `let rawPath = ProcessInfo.processInfo.arguments[0]`; `if let resolved = realpath(rawPath, nil) { binaryPath = String(cString: resolved); free(resolved) } else { binaryPath = (rawPath as NSString).standardizingPath }` (the realpath pointer is malloc'd — must free it). Write a launchd plist (atomically, UTF-8) to `plistPath` with separate keys: `Label = the label`; `ProgramArguments = array [binaryPath, "watch"]`; `RunAtLoad = <true/>`; `KeepAlive = <true/>`; `StandardOutPath = /tmp/elap-daemon.log`; `StandardErrorPath = /tmp/elap-daemon.log` (the standard plist XML preamble/DOCTYPE). Then `runLaunchctl(["bootout", "gui/\(uid)/\(label)"])` to clear any stale instance (ignore errors), then `let (out, err, code) = runLaunchctl(["bootstrap", "gui/\(uid)", plistPath])`; if `code != 0`, `fputs("launchctl bootstrap failed (exit \(code)):\n\(err)\(out)", stderr)` and `throw ExitCode(1)`. On success `print "Daemon installed and started."` then the Binary, Plist, Logs paths and `"The daemon will restart automatically at login."` (`uid = getuid()`).

  - **daemon uninstall** — abstract `"Stop and remove the watch daemon."` `let (out, err, code) = runLaunchctl(["bootout", "gui/\(uid)/\(label)"])`. If `code != 0`: `let combined = out + err`; treat as already-stopped if it contains `"No such process" || "Could not find" || "not loaded"` — if not one of those, `fputs(...) + throw ExitCode(1)`. If `code == 0`, `print("Daemon stopped.")`. Then, if the plist file exists, remove it. Always `print("Daemon uninstalled.")` at the end.

  - **daemon status** — abstract `"Show whether the watch daemon is currently running."` `let (out, _, code) = runLaunchctl(["print", "gui/\(uid)/\(label)"])`. If `code == 0`: find the first output line that, trimmed of whitespace, `.hasPrefix("pid")`; if found `print "Daemon is RUNNING (\(pidLine))."` else `"Daemon is RUNNING."`. If `code != 0`: `print "Daemon is NOT running."` and a hint `"  To install: elap daemon install"`.

### 2.9 Helpers

- `@discardableResult private func runLaunchctl(_ args: [String]) -> (stdout: String, stderr: String, exitCode: Int32)`: create a `Process` with `executableURL = URL(fileURLWithPath: "/bin/launchctl")`, `arguments = args`, `stdout/stderr Pipe()`s; `do { try proc.run() } catch { return ("", error.localizedDescription, 1) }`; `proc.waitUntilExit()`; read both pipes via `readDataToEndOfFile()` decoded UTF-8 (default `""`); return `(out, err, proc.terminationStatus)`.
- `func printErr(_ error: Error)`: write `"Error: \(msg)\n"` to stderr via `fputs`; if `error as? DTError`, use its `errorDescription ?? localizedDescription`, else `error.localizedDescription`.
