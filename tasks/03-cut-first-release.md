# Task 03 — Cut the first release

**Goal:** Create the `v0.1.0` git tag and GitHub release, and capture the source-tarball
URL + sha256 that the formula needs.

**Depends on:** 01, 02

**Files to create/edit:** none (git/GitHub actions; developer runs git commands)

## Steps

> Per project rule, **the developer runs all `git` commands**. Present these for the user to run.

1. Tag and push (run by developer):
   ```sh
   git tag -a v0.1.0 -m "ELAP 0.1.0"
   git push origin v0.1.0
   ```
2. Create the GitHub release (auto-generates a source tarball):
   ```sh
   gh release create v0.1.0 --title "ELAP 0.1.0" --notes-from-tag
   ```
3. Record the source tarball URL (stable, auto-generated by GitHub):
   ```
   https://github.com/LightC0de/elap/archive/refs/tags/v0.1.0.tar.gz
   ```
4. Compute the sha256 the formula will pin:
   ```sh
   curl -fsSL https://github.com/LightC0de/elap/archive/refs/tags/v0.1.0.tar.gz | shasum -a 256
   ```
   Save the hash — task 06 needs it.

## Acceptance criteria

- `gh release view v0.1.0` shows the release.
- The tarball URL downloads, and a sha256 has been captured.

## Notes / gotchas

- The tag version must match `elapVersion` (task 01) and the `CHANGELOG.md` entry (task 02).
- Use the GitHub-generated `archive/refs/tags/*.tar.gz` URL, not a manually uploaded asset, so
  the URL pattern is predictable for the bump automation in task 08.
